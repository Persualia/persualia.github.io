Componentes React Integrados

05 de August, 2025

Por Equipo de Desarrollo

Persualia combina la potencia de Jekyll con la interactividad de React. En este artículo, exploraremos los componentes React que hemos integrado y cómo puedes usarlos.

Arquitectura Híbrida

Nuestra aproximación híbrida nos permite:

  • Sitio estático rápido: Jekyll genera páginas estáticas optimizadas
  • Interactividad selectiva: React añade funcionalidad donde la necesitamos
  • SEO optimizado: El contenido se renderiza en el servidor
  • Hidratación progresiva: Los componentes React se activan cuando es necesario

Componentes Disponibles

1. ThemeSwitcher

El selector de temas es uno de nuestros componentes React más útiles:

const ThemeSwitcher = () => {
  const [currentTheme, setCurrentTheme] = React.useState('persualia');
  
  const handleThemeChange = (theme) => {
    setCurrentTheme(theme);
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('theme', theme);
  };
  
  // ... resto del componente
};

Características:

  • Persistencia en localStorage
  • Cambio instantáneo de tema
  • Lista completa de temas disponibles
  • Interfaz intuitiva con dropdown

Cómo Añadir Nuevos Componentes

Paso 1: Definir el Componente

Agrega tu componente en _includes/react-components.html:

const MiNuevoComponente = ({ prop1, prop2 }) => {
  const [estado, setEstado] = React.useState(null);
  
  return (
    <div className="mi-componente">
      {/* Tu JSX aquí */}
    </div>
  );
};

Paso 2: Registrar el Componente

Añade el código de renderizado:

document.addEventListener('DOMContentLoaded', function() {
  const container = document.getElementById('mi-componente');
  if (container) {
    const prop1 = container.getAttribute('data-prop1');
    const prop2 = container.getAttribute('data-prop2');
    
    ReactDOM.render(
      <MiNuevoComponente prop1={prop1} prop2={prop2} />, 
      container
    );
  }
});

Paso 3: Usar en HTML

<div id="mi-componente" 
     data-prop1="valor1" 
     data-prop2="valor2">
</div>

Mejores Prácticas

Rendimiento

  • Carga bajo demanda: Solo carga React cuando hay componentes que lo necesiten
  • Componentes ligeros: Mantén los componentes simples y enfocados
  • Evita re-renders: Usa React.memo cuando sea apropiado

Mantenibilidad

  • Componentes puros: Prefiere componentes funcionales
  • Props tipadas: Usa PropTypes o TypeScript para validación
  • Documentación clara: Comenta el propósito de cada componente

Integración

  • Datos del servidor: Usa data attributes para pasar datos desde Jekyll
  • Estado global mínimo: Evita state management complejo
  • Fallbacks: Proporciona alternativas si JavaScript está deshabilitado

Conclusión

Los componentes React en Persualia proporcionan la interactividad perfecta sin sacrificar el rendimiento. Esta arquitectura híbrida es ideal para sitios que necesitan ser rápidos pero también interactivos.
